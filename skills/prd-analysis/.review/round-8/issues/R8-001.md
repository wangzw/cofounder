# CR-L11 criteria-internally-consistent — adversarial-reviewer / per-issue-reviser issue_id schema drift — RESOLVED

## Finding

R6-007 (filed round-6, cross-reviewer) flagged a cross-role schema inconsistency: the
canonical issue-file frontmatter schema mandated by `review/cross-reviewer-subagent.md`
(`id: R<N>-<seq>`) was not honoured by the two sibling prompts that write to the same
`<target>/.review/round-<N>/issues/` directory:

- `review/adversarial-reviewer-subagent.md` used `issue_id: <target-slug>-round-<N>-<seq>`
  (e.g. `prd-analysis-round-1-014`).
- `revise/per-issue-reviser-subagent.md` §Meta-Issue Schema used
  `issue_id: <prd-slug>-round-<N>-<next-seq>`.

R7 dispatched revisers against both leaves. Post-revise verification (this dispatch,
R8-V-001) confirms both leaves are now aligned with the canonical schema.

## Evidence

Grep verification across both revised files:

```
$ grep -n "issue_id:" \
    skills/prd-analysis/review/adversarial-reviewer-subagent.md \
    skills/prd-analysis/revise/per-issue-reviser-subagent.md
# (no matches)

$ grep -nE "(slug|target-slug|prd-analysis)-round-<?[N0-9]+>?-<?[a-z0-9]+>?" \
    skills/prd-analysis/review/adversarial-reviewer-subagent.md \
    skills/prd-analysis/revise/per-issue-reviser-subagent.md
# (no matches)
```

Canonical form present in both:

```
# review/adversarial-reviewer-subagent.md
315:Issue ID format: `R<N>-<seq>` (zero-padded 3 digits; e.g. `R3-014`). Continue the same
318:MUST match the frontmatter `id:` value (e.g. file `R3-014.md` contains `id: R3-014`).
328:id: R<N>-<seq>                        # zero-padded 3 digits; matches filename

# revise/per-issue-reviser-subagent.md
228:Issue ID format: `R<N>-<seq>` (zero-padded 3 digits). Continue the same sequence ...
238:id: R<N>-<seq>
```

Both sites now also contain cross-role alignment prose mandating the schema MUST stay
identical across cross-reviewer / adversarial-reviewer / per-issue-reviser, so any future
change must be applied in all three in the same revision — closing the CR-L11 loop that
R6-007 identified.

## Why this closes R6-007

CR-L11 criteria-internally-consistent guards the review-protocol contract. With both
sibling prompts now using `id: R<N>-<seq>` verbatim and naming each other as paired
update sites, the three writers into `.review/round-<N>/issues/` share a single
frontmatter schema. Downstream readers (`aggregate.py`, judge, summarizer) that key on
`id:` will now see every issue regardless of which role wrote it. The filename-to-`id:`
round-trip invariant holds for all three roles.
