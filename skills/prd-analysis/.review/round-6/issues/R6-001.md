# R6-001 — RESOLVED: Observability prescriptive/advisory stance now made explicit in template

## Prior issue (R5-001)

R5-001 (warning, CR-PRD-L05, escalated from writer self-review `R5-W-001`)
documented a tension: the template prescribed four observability topics via
"Writers MUST emit all four topics below" while CR-PRD-L05 (v1.0.0) still only
enforced perf / security / a11y. The concrete risk: a well-formed Observability
subsection is never checked by any reviewer; a missing one still passes
CR-PRD-L05. The issue was filed as `warning` because delivery-2 designed this
narrow-change compromise intentionally; the fix was deferred to delivery-3.

## Resolution evidence (round-6 snapshot of `common/templates/artifact-template.md`)

The reviser (R5-R-001) did NOT change CR-PRD-L05 semantics — correctly, since
that was explicitly out of scope for delivery-2. Instead, the prescriptive/
advisory stance is now called out cleanly and the cross-subsection ownership
map is in place. The tension R5-001 documented is now presented as an
intentional, scope-bounded design rather than a latent gap.

1. **Line 27 — directory-layout comment unchanged and coherent:**

   ```
   │   └── nfr.md                        # perf + security + a11y (CR-PRD-L05) + observability (advisory)
   ```

   The `(advisory)` annotation is intact; no reader is misled about what
   CR-PRD-L05 enforces.

2. **Lines 251-253 — CR-PRD-L05 enforcement prose preserved:**

   > `nfr.md` MUST cover all three non-functional dimensions — performance,
   > security, and a11y — to satisfy CR-PRD-L05. An architecture section
   > missing `nfr.md` or any of these three dimensions fails this criterion.

3. **Lines 255-260 — Observability subsection header explicitly flags the
   advisory status and the future-CR upgrade path:**

   > ### nfr.md — Observability Subsection (Prescriptive)
   >
   > In addition to the three CR-enforced dimensions, every `nfr.md` SHOULD
   > include an **Observability** subsection. Writers MUST emit all four topics
   > below when authoring a PRD's `architecture/nfr.md`. This subsection is
   > currently advisory (not enforced by CR-PRD-L05). Future delivery may
   > promote this to a formal CR (CR-PRD-L07 observability-coverage).

   The "currently advisory (not enforced by CR-PRD-L05)" + explicit
   CR-PRD-L07 forward reference matches R5-001's recommended deferred action
   path 2 (additive CR-PRD-L07 rather than bumping CR-PRD-L05 to v2). The
   template now codifies the deferred plan in-file, which is the round-6
   closure condition for a deferred warning.

4. **Lines 262-275 — new `Scope boundaries for nfr.md` block** clarifies
   which subsection owns which concern, eliminating the reviewer-can't-check
   ambiguity the R5-001 escalation was rooted in.
