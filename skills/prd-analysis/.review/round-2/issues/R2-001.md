# R2-001 — domain-consultant subagent references stale criteria numbering

## Criterion

**CR-L04 criteria-internally-consistent**: "No two criteria in the target's
`review-criteria.md` MUST have `conflicts_with` references that create oscillation-prone
pairs per guide §13.1." More broadly — criteria references across the skill must stay
internally consistent so the convergence judge and downstream consumers can reach a
stable view.

## Evidence

`common/review-criteria.md` (the authoritative criteria file) now defines:

- Structural (script-type): `CR-PRD-S01`, `CR-PRD-S02`, `CR-PRD-S03`, `CR-PRD-S05`,
  `CR-PRD-S06`, `CR-PRD-S07`, `CR-PRD-S08` (7 criteria; `S04` intentionally absent
  and documented in the "About" section).
- Semantic (llm-type): `CR-PRD-L01` (was PRD-S04, reclassified as llm),
  `CR-PRD-L02`, `CR-PRD-L03`, `CR-PRD-L04`, `CR-PRD-L05`, `CR-PRD-L06` (6 criteria).

However, `generate/domain-consultant-subagent.md` still references the pre-reclassification
numbering. The file contains two contradictory blocks:

1. **R-005 informational listing (lines 170–179)** presents the structural criteria as
   `PRD-S01..S08` — including `PRD-S04 feature-files-self-contained`, which no longer
   exists under that ID:

   ```
   - `PRD-S04` feature-files-self-contained — feature files inline data model + conventions; no cross-refs to other feature files
   ```

   `PRD-S04` has been reclassified to `CR-PRD-L01`. This line tells the user that
   structural criterion `PRD-S04` will be applied, but no such criterion exists in
   `review-criteria.md`.

2. **R-006 informational listing (lines 185–191)** presents the LLM criteria as
   `PRD-L01..L05`:

   ```
   - `PRD-L01` feature-to-journey-mapping
   - `PRD-L02` mvp-discipline
   - `PRD-L03` feature-boundaries-clear
   - `PRD-L04` non-functional-requirements-present
   - `PRD-L05` cross-journey-patterns-identified
   ```

   This numbering is off by one everywhere compared to `review-criteria.md`:
   - Actual `CR-PRD-L01` is `feature-files-self-contained` (the reclassified former
     PRD-S04), not `feature-to-journey-mapping`.
   - Actual `CR-PRD-L02` is `feature-to-journey-mapping`.
   - Actual `CR-PRD-L06` is `cross-journey-patterns-identified` (not L05).
   - There are 6 LLM criteria, not 5.

3. **Output contract (lines 303–323)** echoes the same stale numbering in the
   `clarification.yml` template the consultant writes, so any consultant that follows
   this template will emit a clarification file that is internally inconsistent with
   the authoritative criteria file.

## Why this fails CR-L04

The skill has two self-describing sources of truth for its own criteria: the
authoritative definitions file (`common/review-criteria.md`) and the consultant's
informational recitation in the clarification dialogue. They MUST agree.

Currently:
- A user running `--interactive` mode sees the consultant claim `PRD-S04` and
  `PRD-L01..L05` will be applied.
- The actual reviewer dispatches use `CR-PRD-L01..L06` (including the reclassified
  PRD-S04 as `CR-PRD-L01`).
- The consultant's clarification.yml is persisted into `round-0/clarification/` and
  read by the planner; if the planner or downstream summarizer ever cross-checks
  clarification content against active criteria, the ID mismatch will surface as a
  synthetic drift the judge cannot cleanly resolve.

This is precisely the oscillation-prone inconsistency CR-L04 is designed to catch —
criteria references that do not agree across the skill produce a non-deterministic view
of what "pass" means.

## Fix

Update `generate/domain-consultant-subagent.md` in two places:

1. Rewrite the R-005 listing (section "R-005 — Structural review criteria awareness")
   to reflect the post-reclassification numbering: `PRD-S01, PRD-S02, PRD-S03,
   PRD-S05, PRD-S06, PRD-S07, PRD-S08` (7 entries; explicitly note that `PRD-S04`
   has been reclassified as `CR-PRD-L01`).
2. Rewrite the R-006 listing to match the 6-entry LLM set
   (`PRD-L01..PRD-L06`) with `PRD-L01 feature-files-self-contained` as the first entry.
3. Update the `clarification.yml` template (R-005 and R-006 blocks, lines 303–323)
   to match.

Keep the "About" reclassification note style short — one sentence pointing the reader
to `common/review-criteria.md` for the authoritative definitions.
