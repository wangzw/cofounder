# R3-002 — writer-subagent mis-attributes predicates to CR-PRD-L01 / L06 and still cites CR-PRD-S04

## Criterion

**CR-L04 criteria-internally-consistent**: criterion IDs and their associated
predicates must agree across every leaf in the target skill. If the authoritative
`common/review-criteria.md` defines `CR-PRD-L01` as "feature-files-self-contained"
and `CR-PRD-L02` as "feature-to-journey-mapping", every other leaf that names these
IDs MUST either (a) cite them with the same predicate, or (b) not claim them at all.
Mis-attribution produces synthetic drift between what writers report in self-review
and what cross-reviewers actually check — the judge cannot converge.

## Evidence

The round-3 snapshot of `generate/writer-subagent.md`
(sha256 `25531a527387202a950bcc3515734b04929118f66b7d4f14181ba2a8c1efb582`,
374 lines) contains **five** distinct ID↔predicate mismatches against the
authoritative `common/review-criteria.md`:

### Mismatch A — journey leaf "Applicable CRs" (lines 214–216)

```
Applicable CRs: CR-PRD-S03 (J-NNN ID format), CR-PRD-S08 (file ≤ 300 lines),
CR-PRD-L01 (every touchpoint maps to ≥1 feature — the feature-mapping section
satisfies this from the journey side).
```

`CR-PRD-L01` is `feature-files-self-contained` (per `common/review-criteria.md`
lines 149–164), a feature-file property. "Every touchpoint maps to ≥1 feature"
is the journey-side half of `CR-PRD-L02 feature-to-journey-mapping`
(lines 168–181). The parenthetical predicate belongs to L02, not L01.

### Mismatch B — feature leaf "Applicable CRs" (lines 243–248)

```
Applicable CRs: CR-PRD-S02 (F-NNN ID format), CR-PRD-L01 (feature maps to ≥1
journey touchpoint — satisfied by `journey_refs` + Journey Context section),
CR-PRD-S06 (wikilink targets exist ...), CR-PRD-S08 (file ≤ 300 lines),
CR-PRD-L02 (feature-to-journey mapping present on both sides), ...
```

Same mistake: `CR-PRD-L01` is again attached to the feature-to-journey-mapping
predicate. Note that the very next clause correctly attaches the SAME predicate
to `CR-PRD-L02` — so the file itself encodes a direct contradiction (L01 and
L02 both claimed to check feature↔journey mapping, while L01 is really about
self-containment of the feature file's inline data model / conventions).

### Mismatch C — leaf-type matrix row for journey files (line 284)

```
| `journeys/J-*.md` | CR-PRD-S03, CR-PRD-S08, CR-PRD-L01, CR-PRD-L02 | ID
format + size + feature back-ref from journey side |
```

`CR-PRD-L01 feature-files-self-contained` is a feature-file predicate — it
does not apply to journey files. Including `CR-PRD-L01` in the journey row
will cause the writer to emit a self-review row for L01 against a journey
leaf, which the cross-reviewer then has no way to evaluate consistently.

### Mismatch D — stale CR-PRD-S04 reference (line 318)

```
#### BAD — Feature file that cross-references another feature's data model
(CR-PRD-L01 + CR-PRD-S04 violation)
```

`CR-PRD-S04` no longer exists in `common/review-criteria.md` — per the
"About" section's reclassification note and R2-001's fix, the former
`PRD-S04` is now `CR-PRD-L01`. Citing both `CR-PRD-L01 + CR-PRD-S04` as
separate violations is exactly the pre-reclassification numbering R2-001
tried to expunge; the writer file still carries the stale ID.

### Mismatch E — CR-PRD-L06 misapplied to acceptance-criteria positive/negative pairing (lines 292, 313–314)

```
### Acceptance Criteria Examples (CR-PRD-L06 requirement)
...
- AC-01 is a positive (happy-path) scenario — required by CR-PRD-L06.
- AC-02 is a negative (error/edge) scenario — required by CR-PRD-L06.
```

`CR-PRD-L06` is `cross-journey-patterns-identified` (per
`common/review-criteria.md` lines 236–249), which governs the README's
"Cross-Journey Patterns" section. It has **no** text requiring positive
and negative acceptance-criteria scenarios in feature files. The
acceptance-criteria positive/negative requirement is a template rule from
`artifact-template.md`, not a review criterion at all. Attributing it to
`CR-PRD-L06` will cause the writer to emit `CR-PRD-L06: PASS/FAIL` rows
against feature leaves, while the cross-reviewer evaluates L06 only against
README — guaranteed divergence.

## Why this fails CR-L04

The writer, the cross-reviewer, and the revise-loop reviser all read this
file as the source of truth for "which CR applies to which leaf type". When
the writer's applicability list disagrees with the authoritative criteria
file, three downstream consequences follow:

1. The writer's self-review archive (`self-reviews/<trace_id>.md`) emits
   PASS/FAIL rows keyed on predicates that the cross-reviewer does not
   actually check.
2. The cross-reviewer's leaf↔criterion matrix (built from
   `review-criteria.md`) will not match the writer's, so self-review
   FAIL-row handling (guide §11.1) cannot deterministically escalate,
   dismiss, or cascade.
3. The judge's convergence gate relies on "same criterion, same file"
   stability across rounds. Stale criterion IDs cause the judge to see
   phantom persistent issues (or phantom resolutions) purely from ID
   drift, never reaching a stable verdict.

This is the same failure class as R2-001 — ID-vs-predicate drift across
self-describing leaves — but in a different file, which is why it was not
caught by the R2-001 fix.

## Fix

Update `generate/writer-subagent.md`:

1. **Line 215** — replace `CR-PRD-L01 (every touchpoint maps to ≥1 feature ...)`
   with `CR-PRD-L02 (feature-to-journey mapping — journey side: every
   touchpoint references ≥1 feature)`.
2. **Line 243** — remove `CR-PRD-L01 (feature maps to ≥1 journey touchpoint ...)`
   from the feature applicability list (the L02 clause on line 246 already
   covers this predicate correctly). Retain `CR-PRD-L01` only with its true
   predicate: `CR-PRD-L01 (feature-files-self-contained — data model +
   conventions copied inline; no cross-references to other feature files)`.
3. **Line 284** — drop `CR-PRD-L01` from the journey row. Journeys are not
   subject to self-containment (L01 is a feature-file predicate). Keep
   `CR-PRD-S03, CR-PRD-S08, CR-PRD-L02` only.
4. **Line 318** — remove `+ CR-PRD-S04` from the parenthetical. The current
   `review-criteria.md` has no `CR-PRD-S04`; the predicate is fully captured
   by `CR-PRD-L01` alone.
5. **Lines 292, 313, 314** — remove the `CR-PRD-L06 requirement` attribution.
   The positive+negative acceptance-criteria shape is a template rule from
   `common/templates/artifact-template.md`, not a CR. Either rephrase as
   "(template requirement)" or drop the CR tag entirely.

After the fix, re-verify by grepping `PRD-L01` and `PRD-L06` occurrences in
`generate/writer-subagent.md` and confirming every remaining occurrence pairs
with its authoritative predicate.
