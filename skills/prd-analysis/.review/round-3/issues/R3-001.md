# CR-L11 criteria-internally-consistent — stale CR-ID references in writer self-review checklist

## Finding

The writer-subagent's Self-Review checklist (lines 193–200) and BAD-example annotations (lines 360, 376, 379, 383, 395, 401, 410, 416) cite CR IDs that are **semantically mismapped** to the canonical names defined in `common/review-criteria.md`. Because the writer's self-review is the authoritative record the cross-reviewer consumes under §11.1 FAIL-row handling, the mismatch breaks closed-loop criterion tracking (writer marks "CR-L02 self-contained" FAIL → cross-reviewer opens review-criteria.md and finds CR-L02 is actually `scope-discipline-prd-vs-design`, not self-contained).

## Evidence

`common/review-criteria.md` canonical LLM-criteria (authoritative):

| Canonical ID | Canonical name |
|--------------|----------------|
| CR-L01 | self-contained-file |
| CR-L02 | scope-discipline-prd-vs-design |
| CR-L04 | touchpoint-completeness |
| CR-L05 | design-token-semantic-naming |
| CR-L06 | acceptance-criteria-testable |

`generate/writer-subagent.md` (stale mappings — one per line):

```
line 193: CR-L02 self-contained-file          (actual CR-L02 is scope-discipline; self-contained is CR-L01)
line 194: CR-L05 prd-scope-discipline         (actual CR-L05 is design-token-semantic-naming; scope-discipline is CR-L02)
line 195: CR-S10 leaf-size-budget             (no CR-S10 with that name; size budget is CR-S02; CR-S10 is readme-index-consistency)
line 196: CR-S11 id-format                    (actual CR-S11 is frontmatter-required-fields; id-format is CR-S03/CR-S04)

line 198: CR-L08 touchpoint-completeness      (actual CR-L08 is persona-consistency; touchpoint is CR-L04)
line 199: CR-L09 ac-testable                  (actual CR-L09 is nfr-applicability-coverage; ac-testable is CR-L06)
line 200: CR-L10 design-token-semantic        (actual CR-L10 is cross-journey-pattern-addressed; design-token is CR-L05)
```

BAD-example annotations reuse the same stale IDs (lines 360, 376, 379, 383, 395, 401, 410, 416).

## Why this violates CR-L11

CR-L11 criteria-internally-consistent requires that every CR-ID appearing in any sub-agent prompt resolves to the definition in `common/review-criteria.md`. The current writer checklist is an entirely parallel, ghost catalogue of CR IDs with no alignment to the authoritative file. This is a silent-corruption channel: writers self-score PASS against non-existent checks; cross-reviewers consuming the self-review archives have no way to adjudicate FAIL rows because the cited IDs have different semantics in review-criteria.md.

## Suggested remediation

Re-write lines 193–200 and the BAD-example annotations to use the authoritative CR IDs:

```
- CR-L01 self-contained-file: PASS | FAIL — blocker_scope: <value> — note: <reason>
- CR-L02 scope-discipline-prd-vs-design: PASS | FAIL — ...
- CR-S02 leaf-size-budget-300: PASS | FAIL — ...
- CR-S03 feature-id-format (features only) / CR-S04 journey-id-format (journeys only): PASS | FAIL — ...
- CR-S06 kebab-slug-naming: PASS | FAIL — ...
- CR-L04 touchpoint-completeness (journey leaves only): PASS | FAIL — ...
- CR-L06 acceptance-criteria-testable (feature leaves only): PASS | FAIL — ...
- CR-L05 design-token-semantic-naming: PASS | FAIL — ...
```

And propagate the rename to the BAD-example CR annotations.
