# Verify R1-006 — IPC role-mapping table claims `clarification.yml` at round-N but actual write contract is `round-0/clarification/<ts>.yml`

## R1 issue body (verbatim summary)

R1-006 flagged that the Snippet D role-mapping table in `generate/domain-consultant-subagent.md`
line 33 listed the `domain_consultant` final path as `.review/round-<N>/clarification.yml (or
scoped clarification path)` — but the actual write contract elsewhere in the same file (lines
105 and 108) pinned the path to `.review/round-0/clarification/<ISO-timestamp>.yml`. Two
mismatches: (1) `round-<N>` vs `round-0`; (2) flat `clarification.yml` vs
`clarification/<timestamp>.yml` subdirectory.

The R1 author noted the same misleading row appeared in 4 cascade copies (planner, writer,
cross-reviewer, adversarial-reviewer, per-issue-reviser) and explicitly chose to file ONE issue
against the canonical owner (domain-consultant) with the expectation that the cascade fix would
follow.
