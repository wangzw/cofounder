# CR-L01 self-contained-file — artifact-template.md references non-existent prd-analysis.backup/architecture-template.md as load-bearing source

## Finding

The Architecture Topic Template section (lines 242–281) instructs the writer to use a canonical topic skeleton that is "inherited from `prd-analysis.backup/architecture-template.md` and MUST be used verbatim" (line 246) and says "see backup `architecture-template.md` for the exact section list per topic" (line 281). This backup file does not exist in the current skill tree — there is no `prd-analysis.backup/` directory and no `architecture-template.md` file anywhere under `skills/prd-analysis/`. The template is load-bearing (MUST be used verbatim) yet the source it verbatim-copies from is missing.

## Evidence

Filesystem inventory under `skills/prd-analysis/`:

```
$ find skills/prd-analysis -name 'architecture-template.md' -o -name '*.backup'
# (no matches)
```

Template body lines 242–281 (excerpt):

```
line 246: ... inherited from `prd-analysis.backup/architecture-template.md` and MUST be
          used verbatim for:
line 281: ... (see backup `architecture-template.md` for the exact section list per topic).
```

The referenced content — "the exact section list per topic" — is cited as load-bearing for 22 topic file types listed on lines 247–254 (tech-stack, design-tokens, nfr, security, observability, ...). Writers have no concrete section list to author against.

## Why this violates CR-L01

CR-L01 self-contained-file requires the leaf file to contain all context a consuming agent needs. The architecture-topic portion of artifact-template.md delegates its load-bearing content (per-topic section skeleton) to a backup file that does not exist. Writers authoring `architecture/security.md`, `architecture/performance.md`, etc. cannot satisfy the "verbatim" mandate because the source text is unreachable — the leaf template is self-defeating.

## Suggested remediation

Either (a) copy the authoritative per-topic section skeleton INTO `artifact-template.md` as a full subsection per topic (preferred — aligns with the skill's own self-contained principle), OR (b) if the backup file exists outside `skills/prd-analysis/`, move it in-tree and update the reference; OR (c) replace the "MUST be used verbatim" mandate with a per-topic one-line section list authored inline in artifact-template.md. Option (a) is consistent with how the Feature Template (lines 141–212) already inlines its full section list without deferring to a backup.
