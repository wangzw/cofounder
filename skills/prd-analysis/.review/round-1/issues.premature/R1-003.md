# Cross-reviewer prompt role body missing normative discipline

## Evidence

The role-specific body of `review/cross-reviewer-subagent.md` (lines 43-55) contains only DOMAIN_FILL sentinels and TODO markers:

```
# cross-reviewer-subagent — Cross-Reviewer Role for prd-analysis

<!-- DOMAIN_FILL: populated by writer-subagent during round 1 -->

**Role**: `reviewer` / `reviewer_variant: cross` (`V` in trace_id). Read-only against artifact
leaves; write-only to issue files. No user interaction.

<!-- TODO: list the LLM-type criteria (CR-L entries) applicable to this skill's artifacts -->
<!-- TODO: describe the semantic quality checks specific to this artifact domain -->
<!-- TODO: specify skip-set discipline and forced-full override behavior -->
<!-- TODO: describe issue status progression (new/persistent/resolved/regressed) -->
<!-- TODO: specify writer self-review FAIL-row handling: escalate/dismiss/cascade -->
```

CR-L07 requires normative language (`MUST`, `MUST NOT`, `FORBIDDEN`) for hard checks in reviewer prompts; soft language (`try to`, `prefer`, `ideally`) is FORBIDDEN. The current body has zero normative language and zero hard-check language — it is entirely empty of domain guidance.

## Why this fails CR-L07

A cross-reviewer with no normative check procedures cannot produce consistent issue severity classifications. The plan entry (`plan.md` line 48) explicitly calls out the required normative procedures:
- CR-PRD-L02 walk every feature and confirm ≥1 journey touchpoint reference
- CR-PRD-L03 scan for speculative/scope-creep features
- CR-PRD-L04 diff feature boundaries for overlap
- CR-PRD-L05 confirm architecture topics cover perf/security/a11y
- CR-PRD-L06 identify recurring themes across journeys

None of these check procedures is present. Additionally, the skip-set discipline, forced-full override behavior, and writer self-review FAIL-row handling (escalate/dismiss/cascade) — all of which require normative MUST/MUST NOT language — are missing.

## Suggested fix

Populate the role body with:
1. A "Check Procedures" section listing each CR-PRD-L0x with a `MUST` statement describing the walk/diff/scan procedure.
2. "Skip-set discipline" subsection using normative language: "The reviewer MUST read `cross_reviewer_focus` and MUST NOT open leaves in `cross_reviewer_skip`."
3. "Forced-full override" clause: "If `state.yml.forced_full_cross_review: true`, the skip list is effectively empty."
4. "Issue status progression" table (new/persistent/resolved/regressed) with the matching rule.
5. "Writer self-review FAIL-row handling" section listing the 3 actions (escalate / dismiss-with-record / cascade) with `MUST` language; forbid silent ignoring.
6. Replace any soft phrases (the template doesn't yet contain any, but populated body should use only MUST / MUST NOT / FORBIDDEN for hard checks).
