# Cross-reviewer ACK contract body lacks reviewer-specific reinforcement

## Evidence

`review/cross-reviewer-subagent.md` lines 71-77 provide an ACK Format snippet:

```
### ACK Format

OK trace_id=<trace_id> role=reviewer linked_issues=<comma-separated issue IDs or empty>

Return this ACK as the **single and final line** of the Task return. Nothing after it.
```

While the IPC header (lines 1-40) defines "N writes per dispatch" and "one line ACK", the role body has no **reviewer-specific** reinforcement of the Direct-Write + ACK contract. The canonical `skill-forge/review/cross-reviewer-subagent.md` contains a rich reviewer-specific section including:

- Explicit "FORBIDDEN to write to artifact paths — reviewer writes ONLY to `issues/` and `dismissed-fails/`; never to `<target>/<leaf-path>`."
- "FORBIDDEN to include issue content in the Task return — the ACK is one line only."
- "FORBIDDEN to silently ignore writer self-review FAIL rows."
- `source: self-review-escalation` convention for escalated FAILs.
- `CR-META-skip-violation` meta-issue exception.

None of these is in the target file.

## Why this fails CR-L02

CR-L02 (ack-contract-fidelity) severity is critical but for this file the minimum phrases ("Write to final paths", "exactly one line") are present in the header. The gap is that the reviewer-specific enforcement (FORBIDDEN to write to artifact paths, FORBIDDEN to inline issue content) is not in the role body — increasing the risk that a reviewer mid-session conflates issue file writes with artifact writes, or inlines issue content in the ACK. Downgraded to warning in severity field.

## Suggested fix

In the populated role body, add a "Reviewer-specific FORBIDDEN" subsection listing:
- FORBIDDEN to write to artifact paths (reviewer writes only to `.review/round-<N>/issues/` and `.review/round-<N>/dismissed-fails/`).
- FORBIDDEN to include issue body content in the Task return.
- FORBIDDEN to silently ignore writer self-review FAIL rows (must escalate / dismiss / cascade).
- FORBIDDEN to open leaves in `cross_reviewer_skip` unless forced-full override is active.

Mirror the canonical skill-forge reviewer file's "FORBIDDEN (reviewer-specific)" section.
