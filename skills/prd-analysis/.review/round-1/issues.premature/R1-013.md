# in-generate-review.md lacks a PASS/FAIL example demonstrating the line format

## Evidence

`generate/in-generate-review.md` defines the PASS/FAIL line format (lines 24-29):

```
- <CR-ID> <cr-name>: PASS
- <CR-ID> <cr-name>: FAIL — blocker_scope: <value> — note: <one-sentence reason>
```

But provides no concrete example — neither a worked PASS line nor a worked FAIL line with a realistic PRD scenario.

## Why this matters for CR-L06

CR-L06 (writer-prompt-quality-bar) requires ≥ 1 positive + ≥ 1 negative example for writer-facing guidance. While CR-L06 primarily targets the writer prompt itself, `in-generate-review.md` is functionally part of the writer's instruction set (it is the self-review checklist the writer consumes at generation time — see `generate/writer-subagent.md` line 5 which references it). Without an example, writers will invent their own formats, defeating the scripted line-format contract.

Comparison: the self-review archive for R1-W-001 (`.review/round-1/self-reviews/R1-W-001.md` line 46) shows the writer improvised a format:

```
- CR-PRD-S04-reclassification: FAIL — blocker_scope: needs-human-decision — note: PRD-S04 reclassified as CR-PRD-L01 ...
```

That matches the template — good — but the CR-ID (`CR-PRD-S04-reclassification`) is not a declared CR and not tied to a file-type row in the applicability table. The PASS/FAIL format was followed syntactically but the CR-ID discipline slipped because no worked example anchors the expected CR-ID shape.

## Suggested fix

Add a "Worked Example" subsection to `in-generate-review.md`, e.g.:

```
## Worked Example — writing a feature file

For a file of type `features/F-001-login.md`, the applicability row is
`CR-PRD-S02, CR-PRD-S06, CR-PRD-S08, CR-PRD-L01, CR-PRD-L02, CR-PRD-L03, CR-PRD-L04`.
A valid self-review archive entry:

- CR-PRD-S02 features-have-ids: PASS
- CR-PRD-S06 wikilink-targets-exist: PASS
- CR-PRD-S08 leaf-size-within-limit: PASS
- CR-PRD-L01 feature-files-self-contained: PASS
- CR-PRD-L02 feature-to-journey-mapping: FAIL — blocker_scope: cross-artifact-dep — note: journey J-003 referenced but not yet populated in round 1
- CR-PRD-L03 mvp-discipline: PASS
- CR-PRD-L04 feature-boundaries-clear: PASS
```

Include one negative example showing a malformed line (missing `blocker_scope`, or invalid CR-ID).
