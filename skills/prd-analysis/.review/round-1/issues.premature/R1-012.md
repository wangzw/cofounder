# Writer ACK Format snippet present but role body does not reinforce 2-write contract

## Evidence

`generate/writer-subagent.md` lines 60-66 have only a minimal ACK Format:

```
### ACK Format

OK trace_id=<trace_id> role=writer linked_issues=<comma-separated issue IDs or empty> self_review_status=<FULL_PASS|PARTIAL> fail_count=<N>

Return this ACK as the **single and final line** of the Task return. Nothing after it.
```

The IPC header (lines 1-43) defines the writer's 2-write contract (artifact leaf + self-review file), but the role body has no explicit reinforcement. Given the body is DOMAIN_FILL, the populated version may omit the 2-write reminder and thereby leave the writer at risk of:
- Writing only the artifact (skipping the self-review), producing an ACK with `self_review_status=FULL_PASS` that has no backing archive file.
- Inlining self-review content into the artifact (violating artifact nudity, guide §3.9 hard constraint 1).

## Why this fails CR-L02

CR-L02 requires the ACK contract to be enforced in the sub-agent prompt. The header enforces it globally; the role body should reinforce it with writer-specific language:
- "MUST write exactly 2 files: the artifact leaf at `<assigned-path>` AND the self-review archive at `.review/round-<N>/self-reviews/<trace_id>.md`."
- "MUST NOT include self-review content inside the artifact leaf (artifact nudity)."
- "MUST NOT include artifact content in the Task return."

## Suggested fix

In the populated role body, add a "Writer Output Contract" subsection stating the 2-write rule in normative language. Mirror the "writer" row from the IPC role mapping table but expand with writer-specific anti-patterns (artifact nudity violation, missing self-review archive).
