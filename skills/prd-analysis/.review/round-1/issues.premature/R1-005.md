# Adversarial-reviewer Snippet D header is incomplete vs. skill-forge canonical version

## Evidence

The IPC snippet at the top of `review/adversarial-reviewer-subagent.md` (lines 1-40) is a reduced form of Snippet D compared to the skill-forge canonical version in `skill-forge/review/cross-reviewer-subagent.md` (lines 1-86). Missing items:

- No "FORBIDDEN to include generation content in the Task return" for reviewers specifically (it IS present but phrased only as "ACK is one line only").
- No explicit `FAIL` ACK semantics section (collapsed scope: technical failures only).
- No writer-vs-reviewer ACK differentiation — but this is fine because this file is a reviewer prompt.

Actual concrete problem: lines 1-40 omit the `source: self-review-escalation` issue-file convention and the `CR-META-skip-violation` meta-issue exception. More importantly, the snippet fingerprint is `<!-- snippet-d-fingerprint: ipc-ack-v1 -->` which matches, so CR-S08 (script-type fingerprint check) passes; but the **content** of the IPC block is abbreviated vs. canonical.

## Why this fails CR-L02

CR-L02 (ack-contract-fidelity) requires every sub-agent prompt to enforce the ACK contract. The current snippet does say "Write to final paths **in its own sub-session**" and "Task return is **exactly one line**" — this is CR-L02's minimum. On re-reading, the ACK contract's core is present. **Downgrading to warning-severity observation rather than critical failure.**

However, the role body (lines 43-78) has no independent reinforcement of the ACK contract — relying solely on the header. Since the role body is otherwise DOMAIN_FILL stubs with TODOs, there's a risk that when writers populate the body they forget to restate the ACK contract tied to reviewer-specific output (issue files).

## Suggested fix

Either:
1. In the populated role body, add a "Reviewer ACK Contract" subsection reaffirming: "The reviewer MUST write one issue file per issue found to `.review/round-<N>/issues/<id>.md` and MUST return exactly one line ACK; MUST NOT include any issue body content in the Task return."

OR

2. Expand the head IPC block to mirror the full canonical `skill-forge/review/cross-reviewer-subagent.md` Snippet D (including `source: self-review-escalation` and the `CR-META-skip-violation` meta-issue note).

Note: this issue may be a false positive on strict CR-L02 reading — the minimum phrases are present. Reviser may dismiss if they consider the current snippet sufficient. Flagged here for visibility rather than hard failure.
