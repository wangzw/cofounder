# Domain glossary empty — domain-consultant lacks disambiguation vocabulary

## Evidence

`common/domain-glossary.md` (lines 1-14) is an empty DOMAIN_FILL stub:

```
# Domain Glossary — prd-analysis

<!-- DOMAIN_FILL: populated by writer-subagent during round 1 -->

This file defines domain-specific terms used by the domain-consultant sub-agent to disambiguate user intent during the clarification phase.

Each entry should follow the format:

| Term | Definition | Aliases |
|------|-----------|---------|
| `<term>` | `<canonical definition>` | `<comma-separated aliases>` |

<!-- Writer: populate this file with terms specific to this skill's artifact domain -->
```

CR-L05 applicability: while CR-L05 formally targets the artifact template, the self-contained principle generalizes to the domain-glossary — the domain-consultant prompt is expected to reference terms defined here. If the glossary is empty, the consultant prompt either has to inline every definition (bloat) or fail silently when a user uses an alias.

## Why this fails CR-L05 (spirit)

Plan entry (`plan.md` line 23) specifies required terms:
> feature (F-NNN-<slug>.md self-contained spec), journey (J-NNN-<slug>.md touchpoint sequence), touchpoint, architecture topic file, wikilink, tombstone, evolve baseline, self-contained file, product slug, revision log. Include aliases for common user phrasings (e.g., 'user story' → journey, 'epic' → feature cluster, 'spec' → feature file).

None of these is present. The domain-consultant will not be able to translate user phrasings like "user story" or "epic" into canonical `journey` / `feature-cluster` terminology, producing ambiguous `clarification.yml` output.

## Suggested fix

Populate the glossary table with the 10 required terms plus aliases, per plan.md line 23. Each entry should have a one-sentence canonical definition grounded in R-001..R-007 and the CLAUDE.md glossary. Consider also aligning with the repo-level glossary in `/Users/wangzw/workspace/cofounder/CLAUDE.md` (touchpoint, cross-journey pattern, self-contained file, tombstone) to avoid terminology drift.
