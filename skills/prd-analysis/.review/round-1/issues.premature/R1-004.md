# Adversarial-reviewer prompt role body missing normative attack angles

## Evidence

The role-specific body of `review/adversarial-reviewer-subagent.md` (lines 43-54) is a stub:

```
# adversarial-reviewer-subagent — Adversarial Reviewer Role for prd-analysis

<!-- DOMAIN_FILL: populated by writer-subagent during round 1 -->

**Role**: `reviewer` / `reviewer_variant: adversarial` (`V` in trace_id). Fires ADDITIONALLY
to the cross-reviewer when critical/error issues are found. Hunts for structural anti-patterns
specific to this skill's artifact domain.

<!-- TODO: define the attack angles specific to prd-analysis's artifact type -->
<!-- TODO: list the most likely failure modes introduced by this skill's generators -->
<!-- TODO: specify trigger conditions (must check state.yml adversarial_review_triggered) -->
<!-- TODO: describe no-op ACK behavior when trigger flag is absent -->
```

No attack angles, no failure-mode enumeration, no trigger semantics. CR-L07 requires normative `MUST` / `FORBIDDEN` language for the hard checks the reviewer performs.

## Why this fails CR-L07

The plan entry (`plan.md` line 53) lists 5 specific attack angles that need `MUST hunt for…` language:
1. Features that silently cross-reference data models defined in other feature files (CR-PRD-L01).
2. Hidden MVP scope-creep / speculative v2 sections (CR-PRD-L03).
3. Orphaned journeys / orphaned features (CR-PRD-L02).
4. Architecture-index drift vs. topic-file tree (CR-PRD-S05).
5. For `--evolve` output, mutation of predecessor baseline (R-007 immutability).

None of these is present. Also missing: the trigger clause (`MUST read state.yml.adversarial_review_triggered`; if false, emit no-op ACK with `linked_issues=` empty).

## Suggested fix

Populate the role body with:
1. "Trigger" section using `MUST` / `MUST NOT`: "The reviewer MUST read `state.yml.adversarial_review_triggered`. If absent or false, MUST return the ACK with empty `linked_issues` and MUST NOT open any leaf."
2. "Attack Angles" section — one `MUST hunt for …` line per attack from the plan (5 angles).
3. "Failure Modes" subsection listing common writer shortcuts that produce each attack-angle's anti-pattern.
4. No soft language (`try to`, `prefer`, `ideally`).
