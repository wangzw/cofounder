# in-generate-review.md references skill-forge CRs instead of PRD-domain CRs

## Evidence

`generate/in-generate-review.md` (lines 13-21) still carries the skeleton's generic placeholder table that references skill-forge's own CR identifiers, not the PRD-domain CRs defined in `common/review-criteria.md`:

```
| File type | Applies CRs | Severity floor |
|-----------|------------|----------------|
| `SKILL.md` | CR-S01, CR-S02, CR-S09, CR-L01 | critical for CR-S01/S02/S09 |
| `*-subagent.md` | CR-S08, CR-L01, CR-L02 | critical for CR-S08 |
| `common/review-criteria.md` | CR-S05, CR-L03 | error |
| `common/config.yml` | CR-S06 | error |
| Any artifact leaf | CR-L02 | error |
```

These CR IDs (`CR-S01`, `CR-S02`, `CR-L01`, etc.) are the **skill-forge self-review criteria** used to review the generative-skill scaffold itself — NOT the PRD artifact CRs (`CR-PRD-S01..S08`, `CR-PRD-L01..L06`) the writer should use when producing PRD leaves.

The role of `in-generate-review.md` per plan.md line 43:
> Populate the CR applicability table mapping PRD artifact file types to CR subsets. Rows: README.md (CR-S01 frontmatter, CR-S05 index-consistency, CR-L05 cross-journey-patterns), journeys/J-*.md (CR-S03 IDs, CR-S08 leaf-size, CR-L01 feature-mapping back-ref) …

Plan intent: use PRD-domain CRs. Current file has the scaffold's meta-review CRs — a category confusion.

## Why this fails CR-L09

CR-L09 requires writer self-review discipline — specifically the 4 blocker_scope values AND accurate applicability guidance. While the 4 values ARE listed in `in-generate-review.md` lines 35-40 (PASS), the applicability table routes writers toward the wrong CR universe. A writer self-reviewing a PRD feature file will scan skill-forge CR-S01..CR-L02 (SKILL.md and sub-agent-prompt criteria) instead of `CR-PRD-S02 features-have-ids` / `CR-PRD-L01 feature-files-self-contained` / `CR-PRD-L02 feature-to-journey-mapping` — producing meaningless self-reviews (all N/A) and missing real domain checks.

## Suggested fix

Replace the placeholder applicability table with PRD-domain rows:

| File type | Applies CRs | Severity floor |
|-----------|------------|----------------|
| `README.md` | CR-PRD-S01, CR-PRD-S05, CR-PRD-L06 | error |
| `journeys/J-*.md` | CR-PRD-S03, CR-PRD-S08, CR-PRD-L02 | error |
| `features/F-*.md` | CR-PRD-S02, CR-PRD-S06, CR-PRD-S08, CR-PRD-L01, CR-PRD-L02, CR-PRD-L03, CR-PRD-L04 | error |
| `architecture.md` | CR-PRD-S05 | error |
| `architecture/<topic>.md` | CR-PRD-S06, CR-PRD-L05 | error |
| `REVISIONS.md` | CR-PRD-S07 | warning |

Rationale: each row reflects the CRs that mechanically or semantically apply to the content of the named leaf. Cross-check against `common/review-criteria.md` to keep IDs in sync.
