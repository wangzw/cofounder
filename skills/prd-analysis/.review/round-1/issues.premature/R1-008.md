# Artifact template is empty — cannot evaluate self-contained principle

## Evidence

`common/templates/artifact-template.md` (lines 1-22) is a DOMAIN_FILL stub:

```
# Artifact Template — prd-analysis

<!-- DOMAIN_FILL: populated by writer-subagent during round 1 -->

This file defines the canonical structure for artifacts produced by prd-analysis.
The writer sub-agent uses this template as a structural scaffold when authoring new artifact files.

## Artifact Structure

<!-- Writer: describe the artifact's section layout, required fields, and optional sections -->

## Required Fields

<!-- Writer: list all mandatory sections/fields with their expected content type -->

## Optional Sections

<!-- Writer: list optional sections and the conditions under which they apply -->

## Example

<!-- Writer: provide a minimal valid example artifact conforming to this template -->
```

CR-L05 (artifact-template-self-contained) requires the artifact template to enforce: "feature and module files MUST NOT contain cross-references to other files. All referenced context (data models, conventions, journey context) MUST be copied inline."

## Why this fails CR-L05

An empty template cannot enforce the self-contained principle. Writers populating feature files will have no structural scaffold that demonstrates inlined data-model / conventions / journey-context sections, so the default behavior is likely to be cross-references by path — exactly what CR-L05 and CR-PRD-L01 forbid.

Plan entry (`plan.md` line 28) explicitly requires:
> features/F-NNN-<slug>.md (self-contained spec with inline data model, conventions, journey backreferences, acceptance criteria — MUST be independently readable per R-003)

## Suggested fix

Populate the template with distinct sub-sections for each leaf type:
1. `README.md` structure: frontmatter (title, product_name, date, stakeholders) + feature/journey tables + cross-journey-patterns section.
2. `journeys/J-NNN-<slug>.md` structure: persona, touchpoint sequence, pain points, feature-mapping back-refs.
3. `features/F-NNN-<slug>.md` structure with explicit **inlined** sub-sections:
   - Inline data-model fragment (do NOT reference `architecture/data-model.md`).
   - Inline applicable conventions (copy from `architecture/coding-conventions.md`, do NOT link).
   - Inline journey context summary for back-referenced touchpoints.
   - Acceptance criteria.
4. `architecture.md` index format (~50-80 lines).
5. `architecture/<topic>.md` free-form per-topic structure.
6. An explicit "Self-Contained Principle" clause: "Feature files MUST NOT link by path to any other feature file or architecture topic; all context MUST be copied inline."
7. At least one minimal valid feature-file example demonstrating inlined data model.
8. Enforce leaf-size ≤ 300 lines (CR-PRD-S08).
