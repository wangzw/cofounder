# Writer self-review checklist cites CR-IDs whose names don't match `common/review-criteria.md`

## Offending text (verbatim)

Lines 240-244 of `generate/writer-subagent.md` (Self-Review Archive content structure):

```
- CR-S08 ipc-footer-present: PASS | FAIL — blocker_scope: <value> — note: <reason>
- CR-L01 orchestrator-pure-dispatch: PASS | FAIL — ...
- CR-L02 ack-contract-fidelity: PASS | FAIL — ...
- CR-L05 artifact-template-self-contained: PASS | FAIL — ...
- CR-L06 writer-prompt-quality-bar: PASS | FAIL — ...
```

## Why this fails

Cross-checked against `common/review-criteria.md`, the canonical CR-ID-to-name mapping for prd-analysis is:

| CR-ID | Canonical name (from review-criteria.md) | Name used in writer-subagent.md self-review checklist |
|-------|------------------------------------------|------------------------------------------------------|
| CR-S08 | `criteria-yaml-shape` | `ipc-footer-present` (WRONG) |
| CR-L01 | `scope-discipline` | `orchestrator-pure-dispatch` (WRONG) |
| CR-L02 | `journey-feature-coverage` | `ack-contract-fidelity` (WRONG) |
| CR-L05 | `mvp-discipline` | `artifact-template-self-contained` (WRONG) |
| CR-L06 | `ambiguity-elimination` | `writer-prompt-quality-bar` (WRONG) |

Every name on the checklist names a different criterion than the ID claims. The names listed in the checklist appear to be **skill-forge** meta-criteria (about the correctness of the skill itself), but the IDs collide with prd-analysis's PRD-content criteria. This is a namespace conflict that will confuse every writer dispatch:

- A writer who looks up `CR-L01` in `common/review-criteria.md` finds `scope-discipline` (PRD scope), not `orchestrator-pure-dispatch`.
- run-checkers.sh and the cross-reviewer match by CR-ID literal string; a self-review row "CR-L01 orchestrator-pure-dispatch: PASS" will be matched to the PRD-content `scope-discipline` criterion, producing false-green checks and breaking persistence-tracking across rounds.

This bug is observable in **every R1-W-NNN self-review** produced this round (e.g. `R1-W-001` line "CR-S01 skill-md-frontmatter: PASS" — `CR-S01` is `artifact-pyramid-shape` in review-criteria.md, not `skill-md-frontmatter`). Writers copied the wrong-namespace template directly, and the self-review FAIL/PASS results cannot be reliably matched against the canonical CR registry.

## Class-based scan results

Single source of truth — only writer-subagent.md defines this checklist template. All 10 R1-W-* self-reviews in `.review/round-1/self-reviews/` inherit the broken namespace; they are not focus leaves so I do not file separate issues against them, but they are evidence of the propagation.

## Suggested fix

Two options:

**Option A (recommended)**: Use the canonical prd-analysis CR-IDs and their canonical names from `common/review-criteria.md`, scoped to the leaf-kind being reviewed. Replace the checklist with the leaf-kind-applicable subset, e.g. for a feature leaf:

```
- CR-S02 leaf-size-cap: PASS | FAIL — ...
- CR-S03 id-format-and-uniqueness: PASS | FAIL — ...
- CR-S05 self-contained-discipline-headers: PASS | FAIL — ...
- CR-L01 scope-discipline: PASS | FAIL — ...
- CR-L04 self-contained-readability: PASS | FAIL — ...
- CR-L06 ambiguity-elimination: PASS | FAIL — ...
- CR-L10 testability-ac-observable: PASS | FAIL — ...
- CR-L12 authorization-edge-case: PASS | FAIL — ...
```

**Option B**: If the intent really is for the writer to self-check against skill-forge meta-criteria (a different namespace), introduce explicit ID prefixes (e.g. `SF-CR-L01 orchestrator-pure-dispatch`, `PRD-CR-L01 scope-discipline`) so the two namespaces don't collide. This requires a parallel update to `common/review-criteria.md` and to `generate/in-generate-review.md`'s applicability table.

Option A is simpler and aligns with the `--review` mode's stated criteria source.
