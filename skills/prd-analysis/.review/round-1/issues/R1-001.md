# Stale CR-ID format in SKILL.md (dashed `CR-S-NNN` / `CR-L-NNN`)

## Offending text (verbatim)

Line 13 of `SKILL.md`:

> "...iterative review/revise rounds driven by 5 script-type checkers (CR-S-001..CR-S-005) and 6 LLM-type checkers (CR-L-001..CR-L-006), with a summarizer and judge closing each round."

Line 21 of `SKILL.md` (mode-routing table, `review` row):

> "Script-type checks (CR-S-001..005) + LLM cross/adversarial review (CR-L-001..006); issues filed under `.review/round-<N>/issues/`"

## Why this fails

`common/review-criteria.md` defines criteria with the canonical ID format `CR-S01`..`CR-S08` and `CR-L01`..`CR-L16` (no hyphen between the type letter and the digits). The dashed form `CR-S-001` and `CR-L-001` does not match any defined criterion ID. Downstream agents (cross-reviewer, adversarial-reviewer, run-checkers.sh) match criterion IDs by literal string; a stale ID causes false-green review rounds and prevents the issue body from linking back to a real CR. Per the cross-reviewer spec: "FORBIDDEN to reference CR IDs not defined in `common/review-criteria.md`".

This is also a count error: review-criteria.md defines **8 script-type** and **16 LLM-type** criteria, not 5 and 6 respectively. Both the format and the count claim are stale.

## Class-based scan results

The dashed-format pattern `CR-[SL]-[0-9]{3}` was found in 4 files across the focus list (all 4 reported as separate issues — see R1-001, R1-002, R1-003, R1-004 for related instances). This file accounts for 2 of the matches (lines 13 and 21).

## Suggested fix

Replace the stale references with the canonical IDs and the correct counts:

- Line 13: change "5 script-type checkers (CR-S-001..CR-S-005) and 6 LLM-type checkers (CR-L-001..CR-L-006)" to "8 script-type checkers (CR-S01..CR-S08) and 16 LLM-type checkers (CR-L01..CR-L16)".
- Line 21: change "(CR-S-001..005) + LLM cross/adversarial review (CR-L-001..006)" to "(CR-S01..CR-S08) + LLM cross/adversarial review (CR-L01..CR-L16)".
