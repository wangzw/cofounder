# CR-L11 criteria-internally-consistent — adversarial reviewer invents CR-L12..CR-L18 not defined in review-criteria.md

## Finding

The adversarial-reviewer-subagent cites eight criterion IDs (CR-L11..CR-L18) in its Attack-Angles and Severity-Guidance tables (lines 153, 171, 191, 208, 227, 244, 262, 281, 308, 328, 335–342). Of these, only `CR-L11` exists in `common/review-criteria.md` — and there it is defined as "criteria-internally-consistent", NOT "prd-vs-system-design-scope" as the adversarial reviewer uses it. CR-L12 through CR-L18 are not defined anywhere in `common/review-criteria.md`.

## Evidence

Canonical criteria set (from `common/review-criteria.md` — exhaustive for LLM type):

- CR-L01 self-contained-file
- CR-L02 scope-discipline-prd-vs-design
- CR-L03 journey-to-feature-coverage
- CR-L04 touchpoint-completeness
- CR-L05 design-token-semantic-naming
- CR-L06 acceptance-criteria-testable
- CR-L07 priority-rationale-present
- CR-L08 persona-consistency
- CR-L09 nfr-applicability-coverage
- CR-L10 cross-journey-pattern-addressed
- CR-L11 criteria-internally-consistent

Adversarial reviewer citations (lines 153, 171, 191, 208, 227, 244, 262, 281):

```
CR-L11 (prd-vs-system-design-scope)        — semantically ≠ canonical CR-L11 "criteria-internally-consistent";
                                              the semantic is canonical CR-L02 "scope-discipline-prd-vs-design"
CR-L12 (acceptance-criteria-testable)      — not defined; canonical is CR-L06
CR-L13 (nfr-testable)                      — not defined; closest canonical is CR-L09 "nfr-applicability-coverage"
CR-L14 (journey-edge-case-coverage)        — not defined
CR-L15 (persona-specification-complete)    — not defined; related canonical is CR-L08 "persona-consistency"
CR-L16 (priority-rationale-present)        — not defined here but canonical CR-L07 has this exact name
CR-L17 (feature-scope-disjoint)            — not defined
CR-L18 (journey-feature-coverage)          — not defined; canonical CR-L03 is "journey-to-feature-coverage"
```

And the frontmatter schema on line 308 explicitly instructs reviewers to use `CR-L11 | CR-L12 | ... | CR-L18` — an ID range that mostly does not exist in the review-criteria source of truth.

## Why this violates CR-L11

The adversarial reviewer will file issues whose `criterion_id` frontmatter value points to an undefined criterion. The reviser, judge, and summarizer — all of which open `common/review-criteria.md` to look up severity/name — will silently misattribute or fail to resolve these IDs. Script checkers (`check-criteria-consistency.sh`, `extract-criteria.sh`) that enumerate CR IDs from review-criteria.md will report 100% of adversarial-filed issues as "unknown criterion." Criteria-internally-consistent (CR-L11) fails at the most literal level: the criterion the adversarial reviewer cites is not in the criteria file.

## Suggested remediation

Either (a) extend `common/review-criteria.md` with the seven missing CRs (new IDs starting at CR-L12, keeping canonical CR-L11 = criteria-internally-consistent untouched), OR (b) re-map the adversarial reviewer's attack angles onto canonical CR IDs that already exist:

```
Attack angle                       Canonical CR
Scope drift                        CR-L02 scope-discipline-prd-vs-design
Vague acceptance criteria          CR-L06 acceptance-criteria-testable
Untestable NFR                     CR-L09 nfr-applicability-coverage (extend definition)
Missing edge-case touchpoint       CR-L04 touchpoint-completeness (extend definition)
Under-specified persona            CR-L08 persona-consistency (extend definition)
Priority rationale gap             CR-L07 priority-rationale-present
Overlapping features               (new CR needed; file additive)
Orphan journey                     CR-L03 journey-to-feature-coverage
```

Option (a) preserves attack-angle granularity but requires a criteria-file delta; option (b) collapses angles under broader canonical CRs. Planner chooses.
