# CR-L11 criteria-internally-consistent — cross-reviewer cites CR-L07 as "language discipline" (canonical CR-L07 is priority-rationale-present) and uses undefined CR-PRD-journey-feature-coverage

## Finding

The cross-reviewer-subagent cites `CR-L07` three times as "language discipline" (lines 229, 319, 389). In `common/review-criteria.md`, CR-L07 is canonically defined as **priority-rationale-present** — a completely different semantic. Additionally, the Positive Example on line 362–385 uses `criterion_id: CR-PRD-journey-feature-coverage` in the issue frontmatter — an ID that does not exist in `common/review-criteria.md` at all. Finally, line 148 states the cross-reviewer consumes `CR-L01..CR-L10` whereas review-criteria.md defines CR-L01..CR-L11 (CR-L11 is missing from the cross-reviewer's own scope declaration).

## Evidence

`common/review-criteria.md` canonical:
- CR-L07 name: `priority-rationale-present`
- CR-L11 name: `criteria-internally-consistent`
- No CR-PRD-* IDs defined anywhere in the file.

`review/cross-reviewer-subagent.md` citations:

```
line 148: "Authoritative definitions for CR-L01..CR-L10 plus PRD-domain LLM-type CRs..."
          Stops at CR-L10, but review-criteria.md defines through CR-L11.
line 229: "(that would violate CR-L07 language discipline)"
          CR-L07 canonical is priority-rationale-present, not language-discipline.
line 319: "Soft language violates CR-L07 language discipline..."
          Same miscitation.
line 362–365: Positive example frontmatter —
          criterion_id: CR-PRD-journey-feature-coverage
          No such CR; canonical is CR-L03 journey-to-feature-coverage.
line 389: "Anti-pattern A — soft language in a hard check (CR-L07 fires)"
          Same miscitation.
```

## Why this violates CR-L11

CR-L11 criteria-internally-consistent mandates a single coherent criterion registry. Cross-reviewer is the role that CONSUMES criteria to emit issue files; if it uses an ID with the wrong name (CR-L07 = priority-rationale-present, not language discipline) downstream tools reading the issue frontmatter will open review-criteria.md, see priority-rationale-present, and be unable to reconcile the issue body (which is about soft-language usage). The same applies to `CR-PRD-journey-feature-coverage` in the positive example — the reviser dispatched for that issue cannot find the CR definition.

## Suggested remediation

1. **Line 229, 319, 389**: either (a) introduce a canonical `CR-L12 language-discipline` to `common/review-criteria.md` and reference it here, or (b) drop the inline criterion citation (the "MUST / MUST NOT / FORBIDDEN" rule is a style guideline, not a reviewable criterion — fold it into the reviewer's own prompt as an instruction, not a CR).
2. **Line 148**: update to `CR-L01..CR-L11` to match the canonical count.
3. **Line 365**: replace `criterion_id: CR-PRD-journey-feature-coverage` with the canonical `CR-L03` (journey-to-feature-coverage). Preserve the body wording — only fix the frontmatter value.
