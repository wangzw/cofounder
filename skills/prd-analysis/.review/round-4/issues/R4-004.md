# CR-L01 self-contained-file — planner references four non-existent template files

## Finding

The planner-subagent prescribes `template:` values for each entry in `plan.add` that writers are expected to follow. The four template paths cited (lines 79, 81, 83, 85, 87, 108, 133, 136, 139, 142, 145) point to files that do not exist under `common/templates/`. Only `artifact-template.md` and `review-readme-template.md` exist. When the writer sub-agent opens `common/templates/prd-template.md` (etc.) per its Input Contract (writer-subagent.md line 104), the read fails and the writer falls back to guessing shape — a direct self-containment break.

## Evidence

Actual files in `common/templates/`:

```
common/templates/artifact-template.md
common/templates/review-readme-template.md
```

Planner entries referencing non-existent templates:

```
line 79:  common/templates/prd-template.md
line 81:  common/templates/journey-template.md
line 83:  common/templates/feature-template.md
line 85:  common/templates/architecture-template.md (index variant)
line 87:  common/templates/architecture-template.md (topic variant)
line 108: common/templates/feature-template.md  (tombstone variant)
line 133: template: "common/templates/prd-template.md"
line 136: template: "common/templates/journey-template.md"
line 139: template: "common/templates/feature-template.md"
line 142: template: "common/templates/architecture-template.md"
line 145: template: "common/templates/architecture-template.md"
```

Cross-ref check: writer-subagent.md (line 104, 124, 433) consistently references `common/templates/artifact-template.md` with section selection via `leaf_kind`. SKILL.md (line 179) also names `artifact-template.md` as the single canonical template.

## Why this violates CR-L01

CR-L01 self-contained-file requires that every file referenced by a writer / coder / reviewer can be opened and read in place. The planner emits `plan.md` entries pointing at four template paths that resolve to 404. A writer following the plan verbatim will fail the Read step and cannot produce the leaf. Agents consuming the plan are no longer self-contained because the needed context (template body) lives on disk at a path that does not exist.

## Suggested remediation

Replace every `common/templates/{prd,journey,feature,architecture}-template.md` occurrence with `common/templates/artifact-template.md`, and add a `template_section:` field so the writer knows which H2 of `artifact-template.md` to follow (writer-subagent already expects `template_section` in plan entries — line 113):

```yaml
add:
  - path: "<prd_dir>/README.md"
    template: "common/templates/artifact-template.md"
    template_section: "§ README Template"
    description: "PRD README index"
  - path: "<prd_dir>/journeys/J-001-<slug>.md"
    template: "common/templates/artifact-template.md"
    template_section: "§ Journey Template"
    description: "..."
  # ... etc for feature, architecture-index, architecture-topic, tombstone
```

This aligns planner with writer-subagent and artifact-template.md.
